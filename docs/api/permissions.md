# 权限与授权

这个页面主要解释一件最容易让用户困惑的事：为什么自定义 AI 地址不能直接用，必须先测试连接。

## 内置域名与自定义域名的区别

共享模块里会先把内置服务商转换成一组固定的 origin pattern。

如果当前接口地址属于这些内置域名之一，权限会被视为已知可用，不需要额外弹授权。

如果不是内置域名，就会走自定义权限判断流程。

## 自定义 AI 域名的授权逻辑

核心流程是：

1. 先把用户输入的地址规范化成完整的 chat completions 地址。
2. 再把它转换成浏览器权限可识别的 origin pattern。
3. 先检查当前是否已经拥有这个域名权限。
4. 如果没有，就在允许的场景下请求权限。

## 为什么“测试连接”很关键

在这个项目里，“测试连接”不只是检查接口通不通，还承担了自定义域名授权的入口作用。

也就是说：

- 你点击测试连接时，插件既会验证配置是否合理；
- 也可能顺手完成该域名的浏览器权限申请。

## 用户侧最常见的报错

### 未获得当前 AI 地址的访问权限

通常说明当前域名还没有完成授权。最直接的处理方式就是回到设置页点一次“测试连接”。

### Please open the extension popup and use Test Connection once...

这类提示通常说明当前动作不是在一个合适的用户手势上下文里触发的，浏览器不允许直接弹权限请求。

## 这部分和哪些浏览器能力有关

- `permissions`
- `host_permissions`
- `optional_host_permissions`
- `downloads`
