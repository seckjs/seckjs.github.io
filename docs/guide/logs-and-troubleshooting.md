# 日志与排错

## 日志页主要看什么

日志页会展示自动抓取相关的状态和课程进度，重点包括：

- 自动抓取结果
- 课程进度
- 作业数量
- 已处理数量
- 跳过数量
- 当前作业名称

## 为什么这部分很重要

对于批量流程来说，用户最关心的通常不是“能不能抓”，而是：

- 现在抓到哪一步了
- 哪门课还没完成
- 为什么有作业被跳过
- 是页面问题还是权限问题

## 常见状态说明

### 自动抓取中

说明后台还在读取课程、作业或提交列表。

### 自动抓取失败

说明某一步出现了错误，常见原因包括页面结构变化、页面失效、访问受限或网络异常。

### 自动抓取完成

说明当前这轮队列处理结束，可以回头看汇总数据和课程进度。

## “跳过异常作业”通常意味着什么

插件在下面这些情况下会跳过当前作业：

- 作业已经删除
- 当前作业页没有访问权限
- 页面已失效
- 提交列表抓取失败

## 排错顺序建议

1. 先看状态提示里给出的原始错误语义。
2. 确认当前页面是否还是有效超星页面。
3. 确认接口配置是否还能正常测试通过。
4. 再回看课程进度里是哪一门课、哪一份作业出了问题。
